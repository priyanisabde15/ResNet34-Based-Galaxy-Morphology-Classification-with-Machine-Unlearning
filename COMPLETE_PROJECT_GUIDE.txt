PROJECT OVERVIEW
================================================================================
A production-ready implementation of machine unlearning techniques for 
correcting mislabeled astronomical data in CNN-based galaxy classification.

Research Paper: "Machine Unlearning in Galaxy Morphology Classification: 
Efficient Correction of Mislabeled Data in CNN-Based Astronomical Models"

Optimized for: NVIDIA RTX 3050 4GB VRAM (works on CPU too)

================================================================================
QUICK START (5 MINUTES)
================================================================================

1. Install dependencies:
   pip install -r requirements.txt

2. Test setup:
   python test_setup.py

3. Run pipeline:
   python main.py

4. Launch web app:
   streamlit run app.py

That's it! The system will auto-generate synthetic data if needed.

================================================================================
MACHINE UNLEARNING METHODS
================================================================================

METHOD 1: GRADIENT ASCENT UNLEARNING
-------------------------------------
Concept: Maximize loss on mislabeled samples to "forget" them
Time: 40-60 seconds
Forget Accuracy: 40-50% (good - lower is better)
Retain Accuracy: 85-88% (preserved)

METHOD 2: FISHER FORGETTING
----------------------------
Concept: Add noise inversely proportional to parameter importance
Time: 30-50 seconds
Forget Accuracy: 50-60% (moderate)
Retain Accuracy: 84-87% (preserved)

METHOD 3: FULL RETRAIN (BASELINE)
----------------------------------
Concept: Retrain from scratch on clean data
Time: 180-240 seconds
Forget Accuracy: 35-45% (best)
Retain Accuracy: 87-90% (best)

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "Out of memory"
SOLUTION: Edit config.py → BATCH_SIZE = 8 (or 4)

PROBLEM: "CUDA not available"
SOLUTION: Install CUDA toolkit or use CPU (automatic fallback)

PROBLEM: "No dataset found"
SOLUTION: System auto-generates synthetic data (no action needed)

PROBLEM: "Slow training"
SOLUTION: Normal on CPU. Use GPU for 10-20x speedup.

PROBLEM: "Module not found"
SOLUTION: pip install -r requirements.txt --upgrade

PROBLEM: "Permission denied" (Linux/Mac)
SOLUTION: chmod +x run_pipeline.sh
